# 小球大乱斗

![截屏2026-07-23 22.54.04](https://pauline.oss-cn-shenzhen.aliyuncs.com/img/%E6%88%AA%E5%B1%8F2026-07-23%2022.54.04.png)

## 背景

2026 年 7 月 21 日，上课无聊之时，同学在打英雄联盟手游，我说现在的 AI 已经非常厉害，他沉迷游戏置若罔闻，于是我先是做了一个粒子动画，他说感觉一般般；于是我说可以做一个你现在在打的 LOL，他不相信，所以我又做了一个很简单的 LOL demo，效果确实一般，画面的美工确实难以一步到位；于是我又想到自己以前很喜欢玩《球球大作战》，但是一方面在个人模式组队的玩家太多，另一方面氪金选项太多影响游戏体验，不如自己做一个来体验一下，关键是自己由被平台定义的普通玩家变成了上帝一样的**规则的制定者**。

于是乎，就有了这个 100 多 KB 的 HTML 小游戏，完全由 Grok 4.5 制作。（顺便一提，现在的 Grok Build 简直是 TUI 天花板～）

## 在线开玩

[https://ball-battle-theta.vercel.app/](https://ball-battle-theta.vercel.app/)

项目由 GPT + Vercel 部署，本来也用了一下 Codex 的 site 功能，但是需要用户通过 GPT 登录才能访问，所以直接改 Vercel 了。

## 功能特点

- 单人竞技和团队模式
- 32 个 AI 对手
- 多种场景主题：霓虹、赛博、糖果、水墨、极光、晴空、宣纸、薄荷
- 多种小球皮肤（彩虹贼好看）
- 分裂、吐球、急停、合球、冲刺、护盾、**磁力**、**膨胀**等操作

## 本地运行

这个项目没有复杂依赖，直接在浏览器中打开 `index.html` 就可以运行。

也可以执行构建命令，把静态页面和 Worker 文件复制到 `dist` 目录：

```bash
npm run build
```

构建后输出：

- `dist/client/index.html`
- `dist/server/index.js`

## 操作说明

**移动控制：**
- `WASD` 或 **鼠标**：控制球的移动方向（360 度任意方向）

**技能操作：**
- `J`：分裂（所有符合条件的球同时分裂）
- `K`：吐球
- `L`：急停
- `G`：合球（一键瞬间合并所有球）
- `I`：冲刺
- `O`：护盾
- `H`：磁力（黑洞模式）
- `N`：膨胀（原地膨胀）

**其他：**
- `M`：静音/开启音效
- `Esc`：暂停/继续
- `R`：死亡后重新开始

## AI 机制

- AI 只有在体积达到玩家 **2 倍**以上时才会发起分裂进攻
- AI 进攻前有 **0.8-2 秒**的延迟，给玩家留出反应时间
- AI 之间有独立的冷却时间，不会同时发起攻击

## 目录结构说明

```
ball-battle/
├── index.html          # 游戏主体（包含所有 CSS/JS/HTML）
├── worker/             # Cloudflare Worker（用于部署）
├── dist/               # 构建输出目录
├── package.json        # 项目配置
├── vercel.json         # Vercel 部署配置
└── README.md           # 说明文档
```

**注意：** 虽然游戏主体只有一个 `index.html`（约 100KB），但项目需要额外的配置文件来支持 Vercel 部署：
- `package.json`：定义构建脚本
- `vercel.json`：配置 Vercel 的构建和输出目录
- `worker/`：Cloudflare Worker 脚本（可选的备用部署方案）
- `dist/`：构建后生成的输出目录

## 部署

仓库包含 `vercel.json`，Vercel 会执行：

```bash
npm run build
```

并使用 `dist/client` 作为输出目录。
